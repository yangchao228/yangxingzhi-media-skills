核心不是更多 prompt，而是把创作动作变成可接力、可回放、可验证的流程资产。

一条主线，覆盖完整创作周期

文昌.skill 把内容生产拆成明确阶段。每个阶段只做自己该做的事，输出可以被下一步继续读取，也可以被用户停下来判断。

01

探脉 / 定题

从热点、领域方向或已有材料里筛出可写、可读、可承接账号定位的题。

02

采证 / 立骨

先补事实、来源和反向证据，再把文章骨架、论点顺序和素材位置确定下来。

03

起稿 / 诊文

生成可继续修改的初稿，并判断问题出在选题、结构、信息密度还是表达。

04

整章 / 配图

把正文整理成适合平台发布的形态，补齐图文节奏、卡片拆分和图片 URL。

05

封面 / 出刊 / 归档

完成标题、封面、摘要、发布检查，并把有长期价值的内容沉淀为素材。

可组合模块，不是散装工具箱

每个 skill 都有独立边界，也能被文昌路由串成链路。这样既能单独调用，也能形成稳定 SOP。

ENTRY

文昌路由

识别平台、阶段和下一步动作。用户不用先理解应该调用哪个技能。

EVIDENCE

文昌采证

把事实、来源、反向数据和可信度变成研究包，降低空泛观点风险。

WRITING

公众号 / 知乎 / 小红书

按平台语境处理选题、正文、卡片化和发布表达，而不是一稿通吃。

QUALITY

诊文 / 整章

先诊断是否值得继续推进，再做删减、重排、强化开头结尾和发布版整理。

VISUAL

卡片 / 配图 / R2

支持图文卡片、长文拆卡、本地图片上传 R2 和 Markdown 链接替换。

ASSET

成书审查 / 归档

把值得保留的内容从发布稿升级为素材库条目，服务后续复用和成书。

用 content_state 做跨阶段接力

文昌.skill 的关键资产不是单次生成结果，而是统一状态对象。每个阶段只更新自己负责的片段，并明确下一步需要人的确认点。

为长期内容系统设计

文昌.skill 服务的是持续创作，而不是一次性生成。它把自媒体写作里最容易丢失的判断、证据、结构和发布经验固定下来，让创作者逐步拥有自己的数字生产资料。

直接用自然语言启动

典型入口不是命令表，而是创作者真实会说的话。文昌路由会判断当前应该先探脉、采证、起稿、诊断还是出刊。
