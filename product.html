Workflow

从热点到发布的一条主线

文昌.skill 的价值不是多几个 prompt，而是把创作过程拆成明确阶段。每个阶段只读必要上下文，只更新自己负责的状态，后续可继续接力。

01

探脉 / 定题

从热点、已有方向或外部文章里筛出可写、可读、能承接账号定位的题。

02

采证 / 立骨

补来源、事实、反向证据和可信度，再确定论点顺序与素材位置。

03

起稿 / 诊文

生成可修改初稿，并判断问题出在选题、结构、信息密度还是表达。

04

整章 / 配图

按平台节奏做删改、排版、卡片拆分和 Markdown 图片 URL 处理。

05

出刊 / 归档

补齐封面、摘要、发布检查，并把高价值内容沉淀为成书素材。

Capability Units

可组合模块，不是散装工具箱

这套系统保留单个 skill 的独立能力，同时通过文昌路由把它们变成可执行链路。适合先在个人工作流里使用，再逐步沉淀为产品案例。

ENTRY

文昌路由

识别平台、阶段和下一步动作。用户说目标，系统决定应该先采证、起稿、诊断还是出刊。

ASSET

Human 3.0素材沉淀

State Handoff

用 content_state 保持上下文边界

每个阶段只接收结构化状态、上一阶段正式输出和用户明确新增材料。这样可以避免长对话漂移，也方便以后拆成真正多 agent。

Human 3.0

为长期个人内容系统设计

文昌.skill 服务的是持续创作。它把写作里的判断、证据、结构和发布经验固定下来，让创作者逐步拥有自己的数字生产资料。

Usage

直接用自然语言启动

典型入口不是命令表，而是创作者真实会说的话。文昌路由会判断当前应该先探脉、采证、起稿、诊断还是出刊。

帮我找 3 个最近能写的 AI 选题。
这个选题先帮我采证，再决定怎么写。
把这个主题做成公众号文章。
这篇稿子先诊断，再排版成发布版。
