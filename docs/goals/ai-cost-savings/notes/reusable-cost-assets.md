## 2. Agent Run Guardrail

用于 Codex / Claude Code / GoalBuddy / 多 Agent 任务。

```yaml
task:
  objective:
  role: Scout | Judge | Worker | PM
  allowed_files:
    - 
  read_scope:
    - 
  verify:
    - 
  stop_if:
    - Need secrets or billing credentials.
    - Need files outside allowed_files.
    - Verification fails twice.
    - Task goal becomes ambiguous.
  cost_guardrails:
    max_tool_loops_before_replan: 2
    search_before_opening_files: true
    summarize_large_outputs_before_reasoning: true
    no_full_regeneration_without_reason: true
```

使用规则：

- Scout 只读，负责找证据。
- Judge 只读，负责定范围和风险。
- Worker 只改允许文件，负责交付完整可验证切片。
- PM 维护 board 和 receipt，避免任务漂移。

## 3. Model Routing Table

| 任务类型 | 默认档位 | 升级条件 | 降本动作 |
| --- | --- | --- | --- |
| 文件列表、格式整理、标签、摘要 | 低推理 | 输出影响发布或提交 | 限制输出长度 |
| 内容改稿、平台适配、局部代码修复 | 中推理 | 多文件风险、事实争议 | 先给差异清单 |
| 架构判断、复杂 bug、数据清理策略 | 高推理 | 影响生产或需要最终拍板 | 先 Scout 缩小证据 |
| 最终审计、合规边界、账单决策 | 高推理 | 无法验证或证据冲突 | 保留人工确认 |
| 卡片/封面/多平台资产 | 中推理 | 视觉方向未定 | 先定结构再生成 |

路由原则：

- 先用低成本步骤缩小问题。
- 高推理只处理真正需要判断的部分。
- 简单任务避免长链工具调用。
- 复杂任务先出计划，再读文件。

## 4. Prompt Cache / Stable Prefix 规则

把稳定信息放前面，把变化信息放后面。

稳定前缀适合放：

- 项目使命和长期方向。
- 输出格式。
- 固定角色边界。
- 禁止事项。
- 常用术语解释。
- 工具使用规则。

动态部分适合放：

- 本轮目标。
- 最新用户选择。
- 当前错误。
- 本轮文件片段。
- 本轮验证结果。

不要放进稳定前缀：

- 日期和临时版本号。
- 一次性路径列表。
- 大段日志。
- 未确认的草稿。
- 已过期的对话历史。

## 5. 内容流水线降本 SOP

适用于文昌.skill、多平台出刊、小红书卡片、公众号/知乎改稿。

```text
1. 定题：只输出候选方向和判断，不生成全文。
2. 采证：只保留关键事实、反向证据、可信度，不贴全量材料。
3. 立骨：先确认结构和读者收益。
4. 起稿：只在结构确认后生成正文。
5. 诊文：只指出必须改的地方。
6. 分发：通过 content_state 接力，避免各平台重新理解。
7. 卡片：先定页数、结构、统一视觉，再逐页生成。
8. 归档：只归档通过审查、有长期沉淀价值的内容。
```

降本检查：

- 是否跳过了不必要的完整重写？
- 是否只把 `content_state` 传给下一阶段？
- 是否把小红书图文控制在必要页数？
- 是否先确认方向，再生成发布资产？

## 7. 可沉淀成 Skill 的方向

### ai-cost-health-check

输入：一段工作记录、任务清单或项目目录。

输出：

- 成本面地图。
- 重上下文风险。
- Agent 工具循环风险。
- 模型路由建议。
- 可模板化任务。
- 需要用户拍板的订阅/账单项。

### context-budget-planner

输入：用户目标和项目路径。

输出：

- 必读文件范围。
- 先搜关键词。
- 可忽略历史。
- 输出上限。
- stop_if。

### content-token-budgeter

输入：内容任务目标、目标平台、已有 `content_state`。

输出：

- 是否需要完整生成。
- 是否只需要改一段。
- 多平台复用策略。
- 卡片页数建议。
- 发布前人工确认点。

## 8. 默认执行口令

以后启动 AI/Codex 成本敏感任务，可以直接用这段：

```text
先帮我做成本敏感执行：先搜索再读文件；先判断范围再生成；简单任务低推理；复杂任务先 Scout 再 Judge；Worker 只能改 allowed_files；工具失败两次就停下来复盘；最后给出本轮省掉的上下文、工具调用或返工风险。
```

## 9. 人保留判断权的节点

这些动作不自动化：

- 取消订阅。
- 删除账单或密钥。
- 降级关键工具。
- 修改生产配置。
- 发布涉及具体价格的内容。
- 将某项成本优化固化为团队规则。

系统可以给证据、清单和建议，最后取舍由用户确认。
