# 公众号长文 → 小绿书卡片组

## 边界

本技能只负责公众号长文的方形卡片化和微信生态二次分发。

- 如果用户要小红书图文 HTML，使用 `redbook-cards`
- 如果用户要小红书图片集合提示词、标题、配文、标签，使用 `xiaohongshu-viral-image-skill-v4`
- 如果用户只是泛泛说“长文转卡片”，且没有平台约束，使用 `long-to-cards`

## 概述

将一篇公众号长文拆解为 3–10 张风格统一的图文卡片，根据文章内容动态决定张数，输出为单个 HTML 文件。用户在浏览器中打开后，对每张卡片逐一截图，即可作为微信图文配图或单独发布。

**卡片类型库（按需组合）：**

| 类型 | 用途 | 适用条件 |
|------|------|----------|
| 封面观点图 | 提炼文章核心观点，作为封面吸引点击 | 必选，始终是第 01 张 |
| 金句卡片 | 文章中有传播力的金句，每张一条 | 有足够金句时选用，最多 4 张 |
| 共鸣短帖 | "你是不是也这样"共鸣结构，促进互动 | 文章有情绪/场景共鸣素材时选用 |
| 观点拆解卡 | 将文章核心论点逐条展开 | 文章论点密度高、金句较少时替代部分金句卡 |
| 数据/对比卡 | 突出文章中的关键数据或对比 | 文章含有值得视觉化的数字或对比时选用 |
| 行动清单 | 文章方法论提炼为可执行步骤 | 文章有明确方法论时选用 |
| 关注引导卡 | 引导读者关注公众号 | 必选，始终是最后一张 |

**张数判断原则：**
- 短文（500 字以内）：3–4 张（封面 + 1–2 张正文 + 引导卡）
- 中等篇幅（500–1500 字）：5–7 张
- 长文（1500 字以上）：7–10 张
- 金句丰富的文章多排金句卡；方法论型文章优先排行动清单和观点拆解卡
- 内容不足时宁可少张，不要凑数稀释质量

## Step 1：内容分析与选卡

阅读文章，先判断篇幅和内容类型，再决定卡片组合，不要急于生成代码。

**第一步：判断张数和卡片组合**

根据文章篇幅和内容特征，从卡片类型库中选取 3–10 张，列出本次的卡片清单（如：封面 + 金句×3 + 共鸣帖 + 行动清单 + 引导卡 = 7 张）。

**第二步：提取各卡片所需素材**

**核心观点**（封面必用）：
- 文章最想说的一句话，直接、有冲击力
- 副标题：补充说明或转折

**金句筛选标准**（有则提取，无则跳过）：
- 控制在两行以内（中文 20 字以内最佳）
- 有画面感或对比感
- 脱离上下文也能独立传播

**共鸣场景提炼**（文章有情绪素材时）：
- 把文章描述的抽象感受，转化为 4 个具体的"你是不是也这样"场景
- 用第二人称，描述具体行为，不要用形容词

**观点拆解**（金句不足时替代）：
- 提炼 3–4 条核心论点，每条一句话，配一句延伸说明

**行动清单**（文章有方法论时）：
- 提炼 3 步，每步一句话标题 + 一句话说明
- 最后一条故意留白或引出下篇，制造连载感

**关注引导卡信息**（首次使用时询问用户）：
- 公众号名称（handle）
- 账号定位一句话（如"探索 Human 3.0——用结构杠杆放大自己"）
- 是否有"下篇预告"可以写入

## Step 2：视觉风格系统

### 卡片规格
- 尺寸：500×500px（正方形，适配微信图文）
- 输出：单个 HTML 文件，所有卡片垂直排列

### 配色方案（固定，保持系列统一感）

| 卡片 | 背景色 | 主色调 | 用途 |
|------|--------|--------|------|
| 封面 | `#141414` 深黑 | `#D4A84B` 金色 | 视觉冲击，奠定调性 |
| 金句 #1 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 温暖、易读 |
| 金句 #2 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，装饰形状作差异 |
| 金句 #3 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，左侧金色竖条装饰 |
| 观点拆解卡 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，序号列表结构 |
| 数据/对比卡 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，数字大字展示 |
| 共鸣帖 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，轻盈感 |
| 行动清单 | `#F7F3EC` 暖米 | `#D4A84B` 金色 | 与系列统一，收尾感 |
| 关注引导 | `#F5F0E8` 暖米 | `#D4A84B` 金色 | 呼应封面，首尾统一 |

### 字体
- 中文：`"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`
- 装饰引号：`Georgia, serif`

### 装饰元素规则
- 封面：视觉层次拉满，装饰元素叠加使用——
  - 右侧两个半透明同心圆（`border-radius: 50%`，`border: 0.5px`，金色低透明度）
  - 左上角或右下角一个大尺寸旋转方框（`rotate: 20–35deg`，`border: 0.5px solid rgba(212,168,75,0.12)`，尺寸 160–200px）
  - 顶部一条细横线分隔眉头信息区（`height: 1px`，`background: rgba(212,168,75,0.25)`）
  - 底部色块条或渐变条强化版心感（可选，`height: 3px`，金色）
- 金句 #2：右上角旋转 45° 的小方框（金色描边，透明度 0.3）
- 金句 #3：左侧 4px 实色竖条（`#D4A84B` 金色）
- 共鸣帖：右上角淡金色小圆点点缀（`background: #D4A84B`，透明度 0.15）
- 行动清单：右下角旋转 15° 的方框（金色描边，透明度 0.2）
- 正文卡装饰颜色统一在 `#D4A84B` 金色及其透明变体范围内
- 所有装饰用 `pointer-events: none`，不干扰内容

### 品牌元素
- 每张卡片右下角：`@公众号名称`（用户提供）
- 每张卡片左下角：系列标签（如 `HUMAN 3.0`，用用户的内容系列名）
- 标签样式：`font-size: 10px; letter-spacing: 0.18em; border: 0.5px solid 主色; padding: 3px 10px; border-radius: 2px`

## Step 3：卡片内容写作要点

### 封面观点图

**视觉结构（从上到下）：**
- 眉头信息行：极小字（10px），灰色，写作者身份或内容背景，拉近感（如"一个工程师父亲的3天调研"）
- 主标题：特大字（48–56px），font-weight 800，2–4 行，核心词用金色 `#D4A84B` 高亮
- 副标题：13–15px，灰色，1–2 行，补充说明或制造悬念
- 点睛句：右下角极小字（9px），颜色极浅（`#3a3a3a`），文章最深刻的一句升华

**主标题写法——必须触发情绪，二选一方向：**

方向 A · 焦虑触发（让读者感到"这说的就是我"）：
- 用数字制造紧迫感：`5岁了，你还在用工业时代思维做AI时代决策`
- 直接说出恐惧：`孩子今天学的技能，等他长大可能已经被AI替代`
- 否定常识：`别让孩子学编程` / `报了编程等于报了焦虑`
- 揭穿盲区：`你一直在问学什么，但从来没问过培养什么人`

方向 B · 情感共鸣（让读者感到被理解）：
- 认同普遍处境：`每个家长都焦虑，但焦虑对了方向吗`
- 第一人称逆转：`我本能想给孩子报编程——后来改了`
- 反问句式：`孩子学了这些，到底有没有用？`
- 给出答案的承诺：`AI时代最重要的不是知识，而是这个`

**标题写作禁忌：**
- 不要平铺直叙，不写"关于XXX的思考"
- 不要堆砌形容词，如"超级重要的育儿干货"
- 不要全句金色高亮，最多高亮 2–4 个字的核心词
- 主标题字数控制在 12–18 字，超过则拆成两行断句

**副标题功能：承接主标题，做一个小转折或补充承诺**
- 如主标题制造焦虑，副标题给出方向："3天调研后，我换了选择"
- 如主标题给出答案，副标题补充背景："一个工程师父亲的真实决策过程"
- 副标题字号不超过主标题 1/3，颜色用 `#666` 或更浅

### 金句卡片
- 大引号（`"`）作为装饰元素，72px，系列主色
- 正文：22px，font-weight 600，2–3 行
- 配文：13px，灰色，补充语境，1–2 行
- 卡号标注（`01 / QUOTE`）作为序号

### 共鸣短帖
- 标题：`你是不是也这样` 或类似句式
- 内容：4 个场景，用金色实心小圆点列举
- 末尾 CTA：`如果有 2 条以上说中了你，评论区告诉我`（适配公众号评论区互动）

### 观点拆解卡
- 用于金句不足但论点密度高的文章，替代部分金句卡
- 标题：文章核心主张一句话
- 内容：3–4 条论点，每条格式为"一句论点标题 + 一句延伸说明"
- 用金色短横线或序号作为列表标记

### 数据/对比卡
- 用于文章含有值得视觉化的数字或对比
- 核心数字放大展示（60–72px），搭配简短说明文字
- 对比结构：左右分列或上下分列，用竖线或色块区隔

### 行动清单
- 3 步，每步：圆形序号 + 标题 + 一句说明
- 末尾引语：文章升华句，`font-style: italic`
- 可在最后一步暗示"下篇"，制造期待

### 关注引导卡
- 头像区：深色圆形 + 账号名首字母（金色）
- 账号名 + 用户提供的定位描述
- 3 条关注理由，用 `→` 箭头列举：
  - 第 1 条：账号内容方向（用用户提供的定位描述，不要自行编写）
  - 第 2 条：下篇预告（必须使用用户在 Step 5 提供的下篇主题，不要自行捏造或补充）
  - 第 3 条：关注引导（`欢迎关注，不错过系列更新`）
- CTA 按钮：深色底 + 白字，写 `搜索「公众号名」关注`，右侧 `↗`

## Step 4：HTML 输出规范

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>小绿书卡片组</title>
  <style>
    /* 通用 */
    * { box-sizing: border-box; margin: 0; padding: 0; }
    body {
      background: #E8E4DE; /* 暖灰背景，让卡片有展示感 */
      font-family: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
      padding: 40px 20px;
    }
    .cards-wrap {
      display: flex;
      flex-direction: column;
      gap: 40px;
      align-items: center;
    }
    .card-unit {
      display: flex;
      flex-direction: column;
      align-items: center;
      gap: 10px;
    }
    .card-label {
      font-size: 11px;
      color: #999;
      letter-spacing: 0.15em;
    }
    .card {
      width: 500px;
      height: 500px;
      position: relative;
      overflow: hidden;
      display: flex;
      flex-direction: column;
    }
  </style>
</head>
<body>
  <div class="cards-wrap">
    <!-- 7 张卡片依次放入 -->
  </div>
</body>
</html>
```

**关键规则：**
- 所有卡片写入同一个 HTML 文件，张数根据内容动态确定（3–10 张）
- 每张卡片的 CSS 类名用 `.c1` `.c2` 等前缀隔离，避免样式冲突
- `position: absolute` 的装饰元素必须加 `pointer-events: none`
- 不使用 emoji，不使用外部字体（避免网络加载失败）
- 输出到 `/mnt/user-data/outputs/xhs_cards.html`

**排版规范（必须遵守）：**
- 所有正文卡（金句、共鸣帖、观点拆解、行动清单）内容垂直居中，用 `display: flex; flex-direction: column; align-items: flex-end; justify-content: center`
- 金句卡文字右对齐（`text-align: right`，`align-items: flex-end`），营造设计感
- 金句卡正文加 `white-space: nowrap` 防止句子在不必要的位置断行；若确实需要换行，在语义断点处手动加 `<br>`，不要在句子中间强行换行
- 底部品牌栏（系列标签 + handle）固定用 `position: absolute; bottom: 22px` 脱离文档流，不占内容区高度
- 内容区设置 `padding-bottom: 50px` 预留底部品牌栏空间，防止文字被遮挡
- 金句卡的大引号装饰居中显示，不左对齐
- 列表类卡片（共鸣帖、观点拆解、行动清单）条目文字左对齐，但整体列表块居中（`margin: 0 auto`，设定 `max-width`）
- 列表类卡片必须加 `justify-content: center` 确保内容垂直居中，不能只靠 `padding-top` 撑位置
- 封面卡例外：标题左对齐保持冲击力，副文案可左对齐

## Step 5：用户确认流程

**首次使用前询问：**
1. 公众号名称（handle，如 `AI生命克劳德`）
2. 系列标签名（如 `HUMAN 3.0`，若无则用公众号名）
3. 账号定位一句话（用于引导卡第 1 条）
4. 下篇主题或预告内容（用于引导卡第 2 条，若暂无则告知，引导卡该条留空或写"敬请期待"）

**生成前告知用户：**
- 说明本次计划生成几张卡片、各是什么类型，让用户确认或调整
- 示例："根据文章篇幅和内容，计划生成 6 张：封面 + 金句×2 + 共鸣帖 + 行动清单 + 引导卡，是否调整？"

**生成后输出：**
- 询问是否需要修改某张卡片的文案或增减卡片类型
- 卡片确认无误后，自动输出配套的图文标题和正文描述（见 Step 6）

## Step 6：图文标题与正文描述

卡片生成完毕后，紧接着输出一套配套的图文发布文案，无需用户额外要求。

### 标题写作要点

- 控制在20字以内，有悬念感或反转感
- 用第一人称，制造真实感（"我花X天……""我把X退了""我改变了……"）
- 不要用感叹号，不要堆砌形容词
- 好标题结构：**具体行为 + 意外结果**，如"我花3天调研，把孩子的编程课退了"

### 正文写作要点

- 总字数200-350字，节奏轻快，不写长篇大论
- 结构：**一句钩子开头 → 铺垫自己的处境/困惑 → 认知转折点 → 核心结论 → 引导保存卡片**
- 用短句，多换行，适配手机阅读节奏
- 结论部分可直接引用文章中最有力的一句话（保持原文，不改写）
- 结尾固定句式：`卡片里有……，保存备用。`，让用户知道卡片是精华所在
- 末尾附上5-6个话题标签，优先用垂直领域标签（如 `#科学教育` `#AI时代`），不要全用泛标签
- 全程不加 emoji 或表情符号

## 常见调整场景

| 用户说 | 操作 |
|--------|------|
| 换一下 handle 名称 | `sed -i 's/旧名/新名/g'` 批量替换 |
| 去掉某条内容 | 定位对应 HTML 元素删除 |
| 某张卡文案改一下 | `str_replace` 精准替换文字内容 |
| 增加一张卡 | 新增对应 CSS 类 + HTML 结构 |
| 风格整体调暗/调亮 | 调整各卡片背景色十六进制值 |
| 重新写一版标题/正文 | 按 Step 6 规范重新生成，不修改卡片 |

---

## 注意事项

- 金句选取时，优先选原文中**已经高度凝练**的句子，不要自己重写改义
- 共鸣帖的场景要**具体**，避免"你是不是很忙"这类泛化表达
- 行动清单每步说明控制在 30 字以内，超过则截短
- 引导卡的"下篇预告"必须使用用户提供的内容，用户未提供时写"敬请期待"，不要自行编写或捏造
- 不要在卡片正文里加 emoji 或表情符号，看起来 AI 感很强
