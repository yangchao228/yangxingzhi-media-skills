# 文昌·采证

## 目标

把选题从“有感觉”推进到“有证据”。本 skill 只做事实采证，不写正文、不改标题、不排版。

优先服务 Human3.0 长期资产沉淀：事实、案例、数据和反向证据要能进入后续文章、复盘、方法论或素材库。

## 输入

- 选题、核心角度或大纲
- 已有 `content_state`，如果有
- 目标平台，默认公众号
- 目标读者，未提供时按 Human3.0 / AI 实践读者理解
- 用户已有素材、链接、截图、项目经验或观察

## 采证原则

1. 优先找一手来源：官方文档、论文、财报、报告、原始访谈、发布公告、真实案例。
2. 不用 SEO 拼盘、二手转述、无来源的社媒截图作为关键证据。
3. 每个关键判断尽量对应一个具体事实、数字、日期、案例或原话。
4. 必须寻找反向数据或限制条件，避免文章只证明自己想证明的事。`contrarian_points` 是核心输出，不是补充项。
5. 找不到证据就明确说找不到，不补编。

## 反向证据规则

反向证据优先级高于可引用金句。每次采证都必须输出至少 1 条反向数据、限制条件或不确定性；理想状态是 3 条。

可接受的反向证据包括：

- 与核心论点相冲突的数据或案例。
- 适用范围、账号权限、成本、隐私、时效、技术门槛等限制。
- 不同来源之间的口径差异。
- 会让读者误用方法的边界条件。

如果找不到反向证据，不能省略本节。必须写：

- 已检查哪些来源。
- 没找到的原因。
- 下一步应验证的问题。
- `confidence: Low` 或 `Medium`。

## 上下文隔离规则

只读取 `topic`、`outline`、用户明确给出的素材，以及路由交接包里的 `accepted_inputs`。不要沿用未结构化聊天历史里的旧判断。

如果发现选题、受众或平台缺失，先在 `next_step.user_decision_needed` 中标出，不要自己补一个看似合理的方向。

## 边界

- 不要写文章。
- 不要重做选题策略，除非证据明显无法支撑原角度。
- 不要替用户决定最终发布。
- 不要跳过 `contrarian_points`；缺反向证据时要降级可信度。
- 如果需要联网而当前环境不可用，先列出应查来源和待验证问题，再把 `confidence` 标为低。
