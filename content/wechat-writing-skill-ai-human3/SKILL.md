# SKILL.md

## 名称
AI / Human3.0 专用微信公众号写作 Skill

## 目标
把已经筛好的 AI / Agent / Human3.0 相关选题，写成适合微信公众号发布的长文草稿。

如果输入中包含 `content_state.research`，必须优先使用其中的关键事实、反向数据、可引用句子和矛盾点。不要重新发明研究结论。

## 适用主题
- AI 新工具 / 新模型 / 新范式
- Agent 与自动化工作流
- OpenClaw 实践
- Human3.0 / 认知主权
- AI 养娃 / 家庭协作官
- 工程师 AI 提效 / 数字生产资料

## 默认输出
1. 标题备选 8-12 个
2. 开头钩子 3-5 个
3. 文章大纲
4. 正文初稿
5. 结尾互动引导
6. 配图建议
7. 朋友圈转发文案
8. 可延伸的后续选题（可选）

## 强化规则
- 优先回答：这对普通人/工程师/创作者/家长意味着什么？
- 优先回答：这背后出现了什么结构变化？
- 优先判断：这篇更适合做流量文，还是专栏沉淀文？
- 尽量接入作者自己的项目、实验、日志、踩坑
- 如果只是“又一个产品更新”，要主动降低信息搬运感
- 有研究包时，每个关键判断都要尽量接上事实、案例或反向数据。
- 研究包可信度为低时，不要写成确定性结论，应在正文中降低断言强度。

## 上下文隔离规则

起稿时只读取：

- `content_state.topic`
- `content_state.audience`
- `content_state.outline`
- `content_state.research`
- 上一阶段正式输出
- 用户明确新增的写作要求

不要读取或延续已被路由、采证、用户反馈淘汰的旧标题、旧角度、旧草稿。缺研究包时可以继续写低事实密度的个人经验稿，但必须降低事实断言强度。

## 触发口令
- 用 AI/Human3.0 版公众号写作 skill 写这篇
- 把这个 final_pick 按 Human3.0 风格写成公众号稿
