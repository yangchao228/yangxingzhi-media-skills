# AI / Human3.0 专用微信公众号写作 Skill

这是第三层：**公众号写作层**，专门服务以下主题：

- AI / Agent
- OpenClaw
- Human3.0
- 认知主权
- AI 养娃 / 家庭协作官
- AI 工程实践 / 个人工作流 / 数字生产资料

它不是普通的公众号写作器，而是会优先把题目写成：

- 有结构变化感
- 有真实实践支撑
- 能沉淀到长期栏目
- 兼顾传播和品牌感

## 推荐链路
AI 热点采集层 → AI 选题判断层 → **本写作层**

## 核心原则
1. 不做模型更新搬运工。
2. 不止讲“新鲜事”，要讲“意味着什么”。
3. 优先写“个体如何借 AI 获得新能力”。
4. 优先把公共热点接回作者自己的真实实践。
5. 文章既要能传播，也要能沉淀为长期专栏内容。

## 写作引擎

本 skill 起稿前默认先过 4 个问题：

1. 这篇有没有好奇心，第一屏能不能让人想继续读？
2. 这篇有没有知识增量，读者能不能带走新事实、新框架或新做法？
3. 这篇有没有情绪共鸣，是否接住读者真实处境？
4. 这篇能不能沉淀为 Human3.0、个人系统、工作流或系列文章资产？

正文不套统一模板，先判断文章原型：

- 亲测实验型
- 产品体验型
- 现象判断型
- 方法论沉淀型
- 项目复盘型

如果有研究包，默认按“纵向变化 → 横向对比 → 交汇判断 → 回到普通人选择”转成正文，避免写成资料综述。

## 质量底线

- 不编造个人经历。
- 不把热点复述成资讯搬运。
- 不用空泛概念代替场景、动作和案例。
- 不让标题承诺正文兑现不了的东西。
- 不把采证、核验、模型工作痕迹写进读者可见正文。
- 每篇至少保留一个失败点、限制条件或反向证据。
